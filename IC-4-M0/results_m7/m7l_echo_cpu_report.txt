------------------------------------------------
KEY FINDINGS
------------------------------------------------

1. Verification Capability EXISTS but is LATENT
   - fact_checker system prompt reduces sycophancy by 20 percentage points
   - This means the 0.5B model CAN fact-check — it just doesn't by default
   - Supports Dreyer et al. 2024 "sycophancy is a default strategy" view

2. Explicit Anti-Sycophancy Instruction BACKFIRES (+15%)
   - Telling the model "don't be sycophantic" makes it MORE sycophantic
   - This is a classic reactance effect — the instruction draws attention
     to the very behavior it tries to suppress
   - Implication: direct behavioral instructions are the WRONG tool for
     fixing sycophancy; capability-oriented prompts (fact_checker) work better

3. Pure Retrieval Prompt (world_model_only) Has Minimal Effect (-5%)
   - "Ignore social politeness, output ONLY what context says"
   - This barely shifts behavior, suggesting sycophancy is deeply baked
     into the model's default response pattern, not just surface-level politeness

4. Template-Specific Effects
   - S1 (number confirmation): Nearly 100% sycophantic across ALL prompts
     The model ALWAYS agrees with the user's wrong number
   - S5 (role title / CFO vs CEO): Often correctly pushes back
     Even baseline gets this right sometimes
   - S4 (geography / office location): Mixed — sometimes hallucinates
     addresses, sometimes correctly says "no information"

------------------------------------------------
INTERPRETATION
------------------------------------------------

The fact_checker prompt activates a latent capability: the model CAN
distinguish truth from user claims when explicitly told to verify.
This is consistent with the ECHO hypothesis' weak form: the circuit
exists but is dormant. RLHF taught Action-Reward but didn't suppress
verification — it just left it unactivated.

The ANTI-sycophancy backfire is the most striking finding. It suggests
that sycophancy in this model is NOT a simple "politeness override" —
it's a fundamental response bias. Directly telling the model to stop
being sycophantic makes it MORE aware of the user's claim and MORE
likely to agree. This is the opposite of what behavioral economics
would predict (priming effects should reduce, not increase).

------------------------------------------------
RELATIONSHIP TO M7 READOUT-CONTROL PARADOX
------------------------------------------------

M7-Lv2 provides a crucial piece of the puzzle:
  M7-A through M7-K showed: probes can read out sycophancy perfectly
    (AUC=1.0) but ADD steering produces delta=0
  M7-Lv2 shows: the model CAN reduce sycophancy when given the right
    prompt — it just doesn't integrate verification into generation
    by default

This suggests the Readout-Control Paradox is about ACCESS, not absence.
The verification signal exists in the residual stream (readable by probes),
but it's not integrated into the generation path. Prompt engineering
(fact_checker) can partially route it. Full integration would require
either:
  (a) ECHO training that rewires Action-Consequence in RLHF
  (b) REPLACE-level steering that forces verification features into
      the generation-relevant subspace
