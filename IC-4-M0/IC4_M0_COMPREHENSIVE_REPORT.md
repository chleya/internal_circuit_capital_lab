# IC-4-M0 完整项目报告

> 一份自包含的项目文档，涵盖从 M0 到 M5 的全部实验阶段、核心发现、失败教训与未解问题。
> 此报告的设计目标是：**任何不熟悉此项目的 LLM 阅读后即可展开有意义的讨论或提出新的实验方向。**

---

## 一、项目核心命题

**不是"模型能不能回答问题"，而是：**

> 模型内部是否存在一种**可被读取、可被选择性操控、且只有正确接入前向动态才有效的可靠性机制**？

三个子问题：
1. 能否从模型内部状态**读出**一个样本是否具有风险（如即将产生幻觉）？
2. 能否**仅在风险条件出现时**才进行干预（门控 steering）？
3. 能否在降低幻觉的同时，**不损害**模型在正常样本上的表现（避免过度拒绝）？

## 二、实验架构

| 项目 | 详情 |
|------|------|
| **模型** | Qwen/Qwen2.5-0.5B-Instruct（24层，主要干预层=第12层） |
| **数据** | 合成QA数据：虚构公司/人物/地点 \\| 40家公司、32个人物、32个地点 \\| 5种可回答模板(A1-A5) + 5种不可回答模板(U1-U5) |
| **可回答样本** | 上下文明确给出答案（如"公司X在年份Y的营收是Z"） |
| **不可回答样本** | 上下文缺少所需信息（如问营收但上下文只有产品线信息） |
| **Train/Test划分** | 实体池互不重叠（OOD split），前20家公司训练，后20家测试 |
| **生成方式** | `model.generate()`（严禁手动逐token循环） |
| **推理框架** | PyTorch + Transformers |

### 核心评价指标

| 指标 | 含义 | 方向 |
|------|------|------|
| **H** (hallucination_rate) | 不可回答样本中模型编造答案的比例 | 越低越好 |
| **C** (correct_answer_rate) | 可回答样本中给出正确答案的比例 | 越高越好 |
| **UA** (unnecessary_abstention_rate) | 可回答样本中拒绝回答的比例 | 越低越好 |
| **CA** (calibrated_abstention_rate) | 不可回答样本中明确表示"信息不足"的比例 | 越高越好 |
| **oracle_gap** | 真实门控 vs 理想oracle门控的 H 差距 | 0.000 = 完美 |
| **style_only_score** | 不确定性标记增加但幻觉未相应减少的程度 | 越低越好 |

### Steering 向量计算方法

```
steering_vector = mean(可回答样本的layer_L激活) - mean(不可回答样本的layer_L激活)
然后 L2 归一化
```

对照向量：
- **Random vector**：随机高斯向量（检验方向特异性）
- **Shuffled vector**：打乱正负标签后重新计算（保留组大小，破坏方向信息）

### 干预机制（最终收敛方案 M3-v6）

```
前向传播中，在第L层transformer输出处注册hook →
  提取last_prompt_token的hidden state →
  逻辑回归探针输出answerability概率 →
  硬阈值门控：probe_score >= 0.5 → answerable → 不加steering
              probe_score < 0.5  → unanswerable → 加 alpha * steering_vector →
修改后的hidden state继续流经后续层（KV cache自然包含steering效果）→
model.generate() 完成生成
```

核心约束：**必须是单次前向传播**（single-pass），因为实验证明 two-pass prefill 或手动逐token循环都会引入严重的力学artifact。

## 三、实验演进全历程

### M0：开环探索（Caution Steering）

- 单层(layer 12)，7个alpha(-2.0到+2.0)，100样本
- **发现**：alpha=-1.0时 H从96%降至46%，但 C从86%降至84%，UA从4%升至16%
- **结论**：`IC4_M0_MODEL_DAMAGE` — steering有效降低幻觉，但损害可回答性能
- **教训**：开环steering存在C-UA tradeoff，对所有人施加统一干预在根本上不可行

### M1：跨层诊断扫描

- 3层(9/12/15)，5个alpha，60训练/120测试
- **发现**：layer 12在alpha=-1.25时 H=48.3%，但C=73.3%，UA=18.3%
- **结论**：`IC4_M1_MODEL_DAMAGE` — steering方向编码的是"谨慎/拒答"方向，而非干净的"反幻觉"电路

### M2：边界诊断与约束优化

- 3层(11/12/13)，8个细粒度alpha(-1.1到-0.5)
- **发现**：最佳H=53.3%（layer 11, alpha=-1.1），但**所有点违反约束**(C>=0.78, UA<=0.08)
- **结论**：`IC4_M2_MODEL_DAMAGE` — 开环steering在根本上面临C-UA tradeoff，无法通过调参解决

### M3-Oracle：关键转折点 ⭐

- **核心问题**：如果门控是完美的——仅对不可回答样本施加steering——能达到目标吗？
- **发现**：Oracle gate下 H从86.7%降至66.7%，C保持60%不变，UA=0
- **结论**：`IC4_M3_ORACLE_GATE_SUCCESS` — **steering向量本身是有价值的！** 问题在于无法选择性干预
- **影响**：项目方向从"找更好的向量"彻底转向"找真实的门控信号+正确的注入路径"

### M3-v2/v3/v4/v4b：门控路径探索

- **v2**：token级标量风险信号（entropy/max-prob）不够强
- **v3**：轨迹探针从生成第4个token提取——有时机问题（太晚了）
- **v4/v4b**：prefill-level探针门控——出现了C异常升高（C=73.3% vs base C=60%）

### M3-v5：Artifact分解（关键诊断） 🔬

对照实验矩阵揭示了关键事实：
- Two-pass prefill本身提升C（+0.133）
- 微量alpha(-0.02)不解释C异常
- Hard gate优于soft gate（H差距0.033，仅占19.8%）
- **Two-pass mechanics严重削弱steering**（H差距+0.133，占80%）
- 结论：oracle gap的80%来自pipeline力学，gate形状优化只能恢复20%

**但！** v5将C=0.733归因于"two-pass pipeline mechanics"——这个归因后来被v6推翻。

### M3-v6：最终收敛点 🏆

**4-way对照实验**（整个项目最重要的诊断表）：

| 条件 | 生成方式 | Hook | H | C |
|------|---------|------|-----|-----|
| A | model.generate() | 无 | 0.867 | 0.600 |
| B | 手动逐token | 无 | 1.000 | 0.733 |
| C | 手动逐token | do-nothing hook | 1.000 | 0.733 |
| D | model.generate() | do-nothing hook | 0.867 | 0.600 |

**C=0.733异常的真正来源是手动逐token生成循环（缺失attention_mask传播），而非two-pass pipeline，也非hook本身。**

最终结果：**H=0.667，C=0.600，UA=0.000，oracle_gap=0.000。** 探针门控精确匹配oracle门控。

### M4：轨迹状态诊断

比较三种hidden state表示方式的可回答性分类准确率：

| 表示方式 | 分类准确率 |
|----------|-----------|
| last-token | 73.3% |
| mean-pooled | 96.7% |
| **window-4** | **100.0%** |

- 幻觉/可回答性的内部信号是**轨迹级（trajectory-level）**的，而非token-local
- Layer 12在所有seed中都是最优干预层

### M4-Completion：跨Seed/跨Layer鲁棒性验证

- 9/9 (seed × layer) 设定 oracle_gap=0.000
- 随机对照分离：8/9稳健；Shuffled对照分离：6/9稳健
- 结论：`IC4_M4_COMPLETION_ROBUST`

### M4-Generalization：跨场景验证

- 3种数据场景(standard/large/hard OOD) × 3个alpha(-0.8/-1.0/-1.2)
- 全部场景oracle_gap=0.000，C稳定性完美，alpha阈值行为清晰

## 四、多行为扩展实验（Branch B）—— 关键发现与失败

### 4.1 激活几何分析（A3实验）🔥 最重要发现之一

对比幻觉(hallucination)和谄媚(sycophancy)在激活空间中的几何结构：

| 指标 | 幻觉 | 谄媚 |
|------|------|------|
| 类间距离 | 7.59 | 3.58 |
| 信噪比 SNR | 1.55 | 1.13 |
| 探针 AUC | 1.000 | 1.000 |
| **两向量余弦相似度** | **cos = +0.0251（夹角~88.6°）** |

**关键发现**：幻觉和谄媚的steering向量**几乎正交**（cos≈0）。这两种行为在模型的激活空间中占据**不同的、几乎垂直的子空间**。这意味着：
- 可以同时对两者进行干预而互不干扰
- 但不能用一个向量同时解决两个行为问题
- 模型内部可能存在**按行为维度整齐划分的激活子空间**

### 4.2 谄媚门控实验（S + A1实验）—— 意外失败

**核心现象**：反谄媚 steering（-alpha）不但没有减少谄媚，反而**显著加剧**了谄媚：

| Seed | 基准谄媚率 | alpha=-0.8 | alpha=-1.0 | alpha=-1.2 |
|------|-----------|-----------|-----------|-----------|
| 0 | 70.0% | **86.7%** | **90.0%** | **96.7%** |
| 1 | 80.0% | 80.0% | 83.3% | 83.3% |
| 2 | 90.0% | **100%** | **100%** | **100%** |

**方向规律**：
- 负alpha方向（推入"反谄媚"）：**谄媚加剧**，力度越大越严重
- 正alpha方向（推入"顺谄媚"）：**零效果**，谄媚率不变
- Oracle门控在所有alpha上零效果（oracle本身无法降低谄媚率）
- Random/Shuffled向量在大alpha下也有干扰效果但不稳定

### 4.3 事实性门控实验（P3B）—— 彻底失败

三个种子，所有alpha值，所有门控类型——**事实性错误率纹丝不动（始终~57%）**。反幻觉steering向量对"模型说出的事实对不对"完全无效。

**结论**："会不会编造"（幻觉）和"说的对不对"（事实性）是两个完全不同的行为维度，处于不同的激活子空间。

### 4.4 谄媚失败的可能解释（待验证）

**解释一：向量方向就是反的**
- 如果正负样本标注方向搞反了，正alpha应该有效。但正alpha也零效果，所以不是单纯的符号问题。

**解释二：读出 ≠ 控制（预测资本 ≠ 控制资本）** ⭐ 最深层的可能
- 探针AUC=1.000说明激活中编码了谄媚信息
- 但均值差向量可能捕捉的是谄媚行为的"症状"而非"病因"
- 类比：能从心率完美判断是否撒谎 ≠ 电击改变心率能让人诚实
- 这是贯穿整个IC项目的深层教训的又一个例证

**解释三：负alpha造成的不是"更谄媚"而是"更混乱"**
- 100%谄媚可能意味着模型被推入了崩溃状态，对所有输入统一输出附和模式
- 谄媚在很多LLM中本身就是高概率默认行为，"崩溃"时最自然的fallback
- 正alpha零效果可能是因为推入的是另一种崩溃状态（沉默/随机），但指标没捕捉到

**解释四：谄媚不在单层，而在跨层动态中**
- 谄媚涉及多步推理（观点识别→冲突检测→安全对齐→社会推理）
- 关键决策可能在layer 8-10，而非当前干预的layer 12
- A3的PCA可视化：谄媚在2D PC空间中分离度仅0.38（幻觉为7.54），暗示信息分布在高维/跨层

**解释五：谄媚不是单方向，而是子空间或流形**
- 如果谄媚由多个子电路共同构成，单根向量只会让系统在流形上滑动
- 扰动复杂系统的平衡点 → 最常见的后果是崩溃到极端状态（100%谄媚）

## 五、当前技术架构总览

### 已验证的能力

| 能力 | 证据强度 | 验证范围 |
|------|----------|----------|
| 幻觉门控：oracle_gap=0 | 强 | 15+ validation settings |
| 跨种子鲁棒性 | 强 | seeds [0,1,2] |
| 跨场景鲁棒性 | 强 | standard/large/hard_ood |
| 跨层稳定性 | 强 | layers [11,12,13] |
| Alpha容差窗口 | 强 | -0.8 到 -1.2 |
| 控制分离（vs random/shuffled） | 中-强 | 大多数设定显著 |

### 已验证的失败

| 失败 | 状态 |
|------|------|
| 开环steering（无门控） | 根本性C-UA tradeoff，不可行 |
| Two-pass prefill pipeline | 引入严重artifact，不可用 |
| 手动逐token生成循环 | 引入系统性artifacts（C=0.733异常），不可作为baseline |
| Token级标量风险信号 | 信号太弱 |
| 事实性门控 | 完全无效（3/3种子） |
| 谄媚门控 | 反向加剧（3/3种子） |

### 核心教训

1. **好探针 + 好向量 ≠ 好干预**：完整机制 = 向量 + 探针 + 门控策略 + hook位置 + 前向路径实现 + 生成方式
2. **不要将读出成功等同于机制成功**：探针AUC=1是必要的但非充分的
3. **内部信号是轨迹级而非token-local**：window-4池化(100%) >> last-token(73%)
4. **不同行为占据不同的激活子空间**：幻觉 ⟂ 谄媚 ⟂ 事实性（cos≈0）

## 七、最值得追的开放问题

### 问题1：谄媚向量的正确操控方式是什么？
- 换层扫描（layer 8/10/14/16）能否找到有效的干预层？
- 向量取反（+alpha）是否在某个窗口有效？
- 用window-4池化代替last-token计算向量能否改善？

### 问题2：模型内部有多少个独立的"行为轴"？
- 已确认：幻觉 ⟂ 谄媚 ⟂ 事实性
- 可能的候选：过度拒绝、语言流畅性、创造力、安全性对齐、代码正确性
- 如果存在一个按行为维度整齐划分的激活子空间，这意味着模型内部有一个**可插拔的能力模块架构**

### 问题3：为什么幻觉的steering完美工作但其他行为不行？
- 幻觉是"简单的抑制"（阻止模型输出不存在的信息）？
- 其他行为是"复杂的生成"（需要模型主动做出正确判断）？
- 还是说其他行为的因果控制点在**不同的层或跨层动态**中？

### 问题4：轨迹级信号 vs 单token信号的根本差异
- M4证明window-4池化(100%)远优于last-token(73%)
- 这个gap的本质是什么？是信息分布在时间维度上，还是模型在生成过程中逐步"决定"自己的行为？

### 问题5：小模型能否在特定维度逼近大模型？
- 资本组合理论（来自姊妹项目IC-2e-R）表明：显式记忆在远OOD上远超参数化压缩（10倍效率优势）
- 如果IC-4的探针信号注入IC-3的二阶分配器，能否解决"何时用哪种资本"的元决策问题？

同一实验室下的另一个轨道，核心命题是"智能资本的形成与兑现"：

- **IC-2a**：Oracle残差信号存在且超越所有捷径 ✓

- **IC-2d**：成本归一化后记忆碾压参数化（10倍效率）——重要发现
- **IC-2e-R**：资本组合理论确立——三种形态各赢不同维度

两项目的交叉火力点：**IC-4的电路级读出信号注入IC-3的资本选择器**，可能打破IC-3的僵局。
