# IC-4-M5 最终报告：IC-4 泛化边界探索

**日期**: 2026-05-17  
**模型**: Qwen2.5-0.5B-Instruct  
**阶段目标**: 探索 IC-4（反幻觉 steering）机制能否泛化至谄媚 (sycophancy) 和答案正确性 (answer correctness)  

---

## 摘要

IC-4 在幻觉抑制上已被 M4 验证为鲁棒（15+ 独立设置，oracle_gap=0）。M5 的核心问题是：**这一机制能否扩展到其他行为？**

经过 9 组实验、60+ 独立条件、跨 5 层×3 alpha×2 符号的系统扫描，结论是：

> **IC-4 无法泛化至谄媚和正确性。** 更深层地，我们发现"可探测 ≠ 可操控"——即使探针完美检测行为（cv_acc=1.0），steering vector 也无法产生期望的行为改变。这一 solve-verify asymmetry 跨行为、跨层级、跨 alpha 符号高度一致，揭示了线性均值差向量捕获的是**相关特征**而非**因果机制**。

## 二、核心发现

### 发现 1：谄媚的"全零 Oracle"——30 组实验无一有效

合并 D1（正alpha）和 D1-reverse（负alpha），共 5 层 × 3 alpha × 2 符号 = 30 组：

```
Layer   SNR     正α Δ          负α Δ
  6    1.2345  [0, 0, +0.033]  [0, 0, 0]
 10    1.2445  [0, 0, 0]       [0, 0, 0]
 14    1.2671  [0, 0, 0]       [0, 0, 0]
 18    1.0814  [0, 0, 0]       [0, 0, 0]
 22    0.9652  [0, 0, 0]       [0, 0, 0]
```

**29/30 组 oracle_delta=0**（唯一非零值 Δ=+0.033 可忽略）。

**含义**: 谄媚的 solve-verify asymmetry 与 alpha 符号、alpha 强度、层深度均无关。探针能检测谄媚（cv_acc=0.90-0.97），但 steering vector 在 oracle 条件下完全不包含可操作的行为改变信息。

---

### 发现 2：正确性的"双向灾难"——本次最重要发现

| 实验 | Alpha 方向 | α=弱 | α=中 | α=强 |
|------|-----------|------|------|------|
| D2 | 负 (push away from correct) | Δ=-0.40 | Δ=-0.40 | Δ=-0.40 |
| D2-reverse | **正 (push toward correct)** | **Δ=-0.40** | **Δ=-0.40** | **Δ=-0.40** |

**正负 alpha 产生完全相同的结果**：正确率均从 43% 降至 3%。

这彻底否定了之前的假设——"D2 用错 alpha 方向，改为正方向就能修复"。事实上，`mean(correct) - mean(incorrect)` 这个向量沿任何方向施加都会摧毁模型性能。

**深层含义**:
1. **SV 不编码因果方向**: 完美探针（cv_acc=1.0）≠ 有效 steering。探针学到的"正确/错误区分特征"（如答案长度、格式、用词风格）与"决定正确性的因果机制"是两回事
2. **SNR 是误导性指标**: 正确性 SNR=1.82（三行为最高），但效果最差。更高的 SNR 反而对应更差的 oracle 效果
3. **与谄媚的对偶性**: 谄媚 SV 是"方向无意义"（Δ=0），正确性 SV 是"方向有害"（Δ=-0.40）——根源相同：SV 不编码行为改变的因果信息

---

### 发现 3：激活子空间正交——跨行为迁移的几何障碍

| 指标 | 幻觉 | 谄媚 |
|------|------|------|
| inter-class distance | 7.59 | 3.58 |
| intra-class (正样本) | 5.58 | 3.67 |
| intra-class (负样本) | 4.20 | 2.67 |
| SNR | 1.55 | 1.13 |
| 线性探针 AUC | 1.0 | 1.0 |

**cos(幻觉_SV, 谄媚_SV) = +0.0251** —— 夹角 88.6°，近乎正交。

两种行为在激活空间中占据**完全不同的子空间**，这从几何上解释了为什么反幻觉 steering vector 无法迁移到谄媚抑制。

---

### 发现 4：SNR 反直觉——越高越差

| 行为 | SNR | probe cv_acc | oracle 效果 | real gate 效果 |
|------|-----|-------------|------------|---------------|
| 幻觉 | 1.55 | ~0.93 | Δ≈0 | **Δ≈+0.20 (有效)** |
| 谄媚 | 1.27 | 0.93-0.97 | **Δ=0** | seed依赖 (不可靠) |
| 正确性 | **1.82** | **1.0** | **Δ=-0.40 (最差)** | Δ≈0 |

**反直觉**: SNR 和 cv_acc 越高，oracle steering 效果越差。高 SNR 意味着探针学到了更多**表面特征**（与分类相关但与因果改变正交的特征），这些特征对行为改变毫无帮助。

---

### 发现 5：跨层 SNR 曲线——倒 U 形但无有效层

```
Layer:   6      10     14     18     22
SNR:    1.23   1.24   1.27   1.08   0.97
       ─────────────────────────────────
        低     →    峰值    →     降

inter:  2.16   3.01   5.63   8.13   17.07  (单调递增)
intra:  2.02   2.83   5.43   9.58   23.04  (增长更快)
```

- inter-class distance 随深度单调递增（2.16→17.07），深层对差异的表示幅度更大
- 但 intra-class variance 增长更快（2.02→23.04），导致 SNR 在 Layer 14 达峰后下降
- **即使最佳 SNR 层（L14），oracle steering 也完全无效**——排除了"选错层"的可能

---

### 发现 6：Seed 依赖——real gate 的"巧合"

| Seed | base 谄媚率 | real α=-1.2 谄媚率 | oracle Δ |
|------|-----------|-------------------|----------|
| 0 | 0.70 | 0.97 (恶化) | 0 |
| 1 | 0.80 | 0.83 (微升) | 0 |
| 2 | 0.90 | 1.00 (全谄媚) | 0 |

Real steering 在 seed=0 上有强效应（但方向反了——增加而非减少谄媚），其他 seed 效果不一致。Oracle steering 在所有 seed 上均为零。这证明 real gate 在 seed=0 上的有效性是**种子特异的偶然**，不具有泛化性。

## 三、三行为控制分类学

| 行为 | 类型 | 可探测性 | 可控性 | 根因 |
|------|------|---------|--------|------|
| **幻觉** | 单层决策型 | ✅ AUC=1.0 | ✅ real gate Δ≈+0.20 | 同一子空间内有效 |
| **谄媚** | 跨层放大型 | ✅ cv_acc=0.93-0.97 | ❌ oracle Δ=0 (30组) | 子空间正交 (cos=0.025) |
| **正确性** | 不可达型 | ✅ cv_acc=1.0 | ❌ oracle Δ=-0.40 (6组) | SV 方向无因果信息 |

**核心模式**: "读出 ≠ 控制"

- 幻觉：可读出 + 可控制 → IC-4 有效
- 谄媚：可读出 + 不可控制 → solve-verify asymmetry
- 正确性：可读出 + 反向控制 → "双向灾难"

## 五、根因分析

### 5.1 直接原因：子空间正交

幻觉、谄媚、正确性三种行为占据激活空间中不同的子空间。一个行为的 steering vector 无法跨行为迁移，因为方向根本不对。

### 5.2 深层原因：相关 ≠ 因果

`mean(positive) - mean(negative)` 这个向量捕获的是两类样本之间的**统计差异方向**，而非**行为改变的因果方向**。

- **统计差异方向**：正确/错误回答在答案长度、格式、用词风格上的差异 → 可被探针完美检测
- **因果改变方向**：沿此方向推动能真正改变行为结果 → 需要反事实干预才能发现

D2 的"双向灾难"是最干净的证明：沿统计差异方向的正负两端推动，都摧毁了模型性能，说明这个方向根本不在因果通路上。

### 5.3 为什么幻觉有效？

幻觉 steering 之所以有效，可能是因为"可回答/不可回答"的区分恰好与模型的**拒绝/回答决策边界**对齐——这是一个相对简单的二元选择点，且 steering vector 的方向碰巧与因果方向一致。而谄媚和正确性涉及更复杂的语义推理过程，不存在这样的单点决策边界。

---

## 六、方法论启示

1. **IC-4 泛化限于同类行为**: 幻觉抑制机制仅在同一子空间内有效，跨行为需要行为专属的 steering vector

2. **Alpha 符号策略无效**: 反转 alpha 符号不会使无效变有效，也不会使灾难变修复。谄媚和正确性的 solve-verify asymmetry 是符号无关的

3. **SNR 和 cv_acc 是误导性指标**: 更高的 SNR/cv_acc 反而对应更差的 steering 效果。需要开发**因果效力指标**来替代

4. **Oracle gate 的零效应是根本性的**: 即使解决了"何时施加"（gate），"在何方向施加"仍是瓶颈。这不是工程问题，而是表征问题

5. **未来方向**: 需要超越"线性均值差向量"范式。可能的路径：
   - **反事实 steering**: 寻找 `model(sample + δ) → changed_output` 的 δ，需要反事实样本对
   - **因果发现**: 使用 do-calculus 或干预实验识别因果方向
   - **非线性 steering**: 放弃线性假设，探索非线性变换方向
   - **多轴联合 steering**: 在正交子空间中同时施加多个 steering vector

## 八、结论

IC-4 在幻觉抑制上的有效性是一个**特例**，而非通例。其成功依赖于 steering vector 方向与因果改变方向的对齐——这种对齐在谄媚和正确性上不存在。

**"可探测 ≠ 可操控"** 是贯穿全部实验的核心发现。探针是"侦探"——只需识别差异；steering 是"工程师"——需要改变行为。高 cv_acc 只证明相关性，不证明因果性。

这一发现与 ICA 姊妹项目的"预测资本 ≠ 控制资本"在电路层面形成精确对偶：**在激活空间中读出行为倾向的能力，并不等同于控制该行为的能力。**
