# P28: Position Separation — L3 编码的不是 token，而是"首次出现"

**日期:** 2026-05-25  
**状态:** ✅ 完成  
**实验:** Priority C+ — 同一 token "funding" 在不同 prompt 位置上的逐层峰值分离验证  
**模型:** Qwen2.5-0.5B-Instruct | **耗时:** 25.7 min | **6 次发生 × 72 passes = 432 passes**

---

## 1. 动机

P27 发现了一个异常：样本 4 中，pos=7 的 "funding" 峰值在 L3 (Δ=+0.486)，而 pos=27 的 "funding" 峰值在 L8 (Δ=+0.137)。

我们需要确认：
- 这是单一异常还是系统性现象？
- 如果系统性的，是 **"位置编码"**（早期 vs 晚期 token 有不同处理层）还是 **"首次出现"**（第一个 occurrence 获得 L3 优先编码，后续被跳过）？
- 第二个 "funding" 的弱效应 (~+0.06) vs 第一个的强效应 (~+0.41) 是否一致？

---

## 2. 实验设计

**协议:** 对每个幻觉样本，分别找到 "funding" 的**所有**出现位置，对每个位置独立运行逐层消融。

| 样本 | 位置 | 首现(funding) #0 | 位置 | 次现(funding) #1 |
|------|------|-----------------|------|-----------------|
| 4-e | 7 | 首现 (context) | 27 | 次现 (question内"total funding") |
| 14-m | 16 | 首现 (context) | 36 | 次现 (question内"total funding") |
| 24-l | 26 | 首现 (context) | 46 | 次现 (question内"total funding") |

```

```

## 4. 解释

### 4.1 修正假设：从 "L3 编码 funding" 到 "L3 编码首次关键语义 token"

P26 的原始结论（L3 编码 "funding"）在技术上是正确的，但不够精确。

更精确的描述：
> **L3 编码的是 prompt 中第一个被识别为关键因果信号的 token。**
> 如果是 "funding"，那就是 "funding"；如果是 "r_and_d_spend"，那就是 "r_and_d_spend"（也接近 L3/L4）。

对于 "funding" 样本：
- **首现（~pos 7-26）:** L3 编码。它是 prompt 中第一个语义上具有域相关性的 token
- **次现（~pos 27-46）:** 被 L3 "跳过"。后续层（L1/L4/L8）以更弱的方式处理——模型已经拿到了概念，不再需要从零编码

### 4.2 类比：Transformer 的 "first-mention bias"

这个发现与 NLP 中已知的 **first-mention bias**（首次提及偏差）高度一致：
- 语言模型中，首次出现的实体获得更强的注意力权重
- 本文首次提供了**逐层级**的证据：首次出现的因果 token 在 L3 获得专属编码，后续出现被 "对账" 而非 "重新编码"

### 4.3 为什么 "r_and_d_spend" 峰值在 L4 而非 L3？

P27 发现 r_and_d_spend（5 个 sub-token）峰值在 L4。

结合 P28 的新理解：
- **L3 用于单 token 首次语义注入**（"funding" 是一个 token，直接识别）
- **L4 用于多 token 首次语义组合**（"r_and_d_spend" 需要 5 个 token 先识别再组合）
- 后续出现无论单 token 还是多 token 都不再被 L3/L4 优先编码

## 5. 对三层瓶颈模型的更新

### 5.1 更新后的 L3 肖像

| 维度 | 旧理解（P26） | 新理解（P27+P28） |
|------|-------------|-----------------|
| 编码对象 | "funding" token | 首个被识别的高 salience 因果 token |
| 编码粒度 | 单 token 级别 | 语义概念首次提及 |
| 后续出现 | 同样编码 | 对账/查重，非重新编码 |
| 位置依赖 | 无 | 仅首次出现，后续被衰减 |

### 5.2 幻觉干预的新启示

**如果首现占据 ~82% 的因果效应，那反幻觉 steering 的最佳注入点是首现位置。**

- 对 "funding" 问题：在 context 中第一个 "funding" 处（~pos 7-26）在 L3 注入
- 对 "r_and_d_spend" 问题：在问题中第一个 "r_and_d_spend" 处（~pos 42-52）在 L4 注入

**不需要处理同一概念的多次出现——首现已承载绝大部分因果信息。**
