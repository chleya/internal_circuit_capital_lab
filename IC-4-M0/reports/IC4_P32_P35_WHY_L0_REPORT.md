## 0. 核心问题

## 1. P32: Embedding-Level Ablation — 嵌入层 vs L0 等价性

### 1.1 实验设计

```
三种干预方式:
  (A) Embedding-only: 在 model.embed_tokens 输出层清零 "funding" 的嵌入向量
  (B) L0-combined:    清零 L0 的 self_attn + mlp 输出 (P30 方式)
  (C) Embedding+L0:   同时清零嵌入层 + L0 的 attn+ffn

关键预测:
  如果 L0 是 embedding→residual 的唯一处理器:
    → Δembed ≈ ΔL0 (两者等价)
    → Δboth ≈ Δembed (L0不增加额外效应，天花板效应)
```

### 1.2 结果

```
Sample │  Δembed   │  ΔL0.comb │  Δboth    │ Δemb/ΔL0 │ Δboth/Δemb │
───────┼───────────┼───────────┼───────────┼──────────┼────────────┤
  4-e  │  +0.3699  │  +0.3993  │  +0.3424  │   0.9x   │   0.93x    │
 14-m  │  +0.3161  │  +0.3251  │  +0.3508  │   1.0x   │   1.11x    │
 24-l  │  +0.2904  │  +0.2917  │  +0.3276  │   1.0x   │   1.13x    │
───────┼───────────┼───────────┼───────────┼──────────┼────────────┤
 MEAN  │  +0.3255  │  +0.3387  │  +0.3403  │   1.0x   │   1.05x    │
```

### 1.3 判决

```
╔═══════════════════════════════════════════════════════════╗
║  Δembed ≈ ΔL0 (1.0x)                                     ║
║  → 嵌入层消融和 L0 联合消融产生几乎相同的效应             ║
║  → L0 是 embedding 层的直接下游处理器                      ║
║                                                           ║
║  Δboth ≈ Δembed (1.05x, ceiling!)                         ║
║  → 在嵌入层消融基础上再加 L0 消融几乎没有额外效应          ║
║  → L0 不是独立的信息编码器，而是 embedding 的传导器        ║
║                                                           ║
║  结论: L0 的工作 = 接收 embedding 输出 → 转换为残差流      ║
║        它不是"产生"信息，而是把 embedding 中的信息"传递"    ║
║        到残差流。这就是为什么 L0 是唯一入口。               ║
╚═══════════════════════════════════════════════════════════╝
```

### 1.4 物理类比

```
Embedding = 水源 (原始 token 信息)
L0        = 水管 (把水从水源引入残差管道系统)
L1-L23    = 管道网络 (残差流传播、混合、增强)

如果你在 Embedding 处切断水源 → 没水
如果你在 L0 处切断水管   → 没水
如果你两处都切断        → 还是没水 (ceiling)

L0 不是水库（它不存储额外信息），它只是引水管道。
这就是为什么 L1-L23 本身不产生新信息——水在 L0 处就已经全部进入系统了。
```

### 2.3 关键发现

```
Head 1: self-attention = 1.000 ★★★★
  → 这个头对 "funding" 的注意力 100% 集中在自身
  → 这是 L0 的 "身份编码头" — 它不仅不看其他 token, 它只看自己
  → 这就是 L0 如何将嵌入信息传递到残差流的机制:
      Head 1 创建一个纯自我参照的残差表示
```

```
Head 分布模式:
  ┌────────────────────────────────────────────┐
  │ Head 1: 纯自聚焦 (1.000)  ← 身份编码       │
  │ Head 11,12,13: 高自聚焦 (0.29-0.38)        │
  │ Head 0,2,5,6,8,9: 中等自聚焦 (0.15-0.29)   │
  │ Head 3,10: 低自聚焦 (0.05-0.07)            │
  │ Head 4,7: 几乎不看自己 (0.004-0.013)        │
  └────────────────────────────────────────────┘
  
  功能分化:
  - Head 1: 身份编码 (100% self)
  - Head 4,7: 上下文聚合 (关注邻居 token)
  - Head 3,10: 混合 (BOS+邻居)
```

### 2.4 为什么 L1-L23 不产生新信息？

L0 的 Head 1 已经创建了因果 token 的**纯自我参照表示**（100% self-attention）。这个表示进入残差流后，L1-L23 看到的已经是一个"打包好的"token 表示。它们不需要重新编码——只需要在残差流中传播和混合。

**这就是为什么 combined ablation 在 L1-L23 几乎为零**：L1-L23 没有"新编码"的工作要做，它们只是在传递 L0 已经编好的表示。

## 3. P34: Tokenization Decomposition — 为什么 r_and_d_spend 效应更小

### 3.1 实验结果

```
Token: funding (1 sub-token: Ġfunding)
    消融1/1 → Δ = +0.3993  ← 一步到位

Token: r_and_d_spend (5 sub-tokens: Ġr, _and, _d, _s, pend)
    消融1/5 (Ġr)     → Δ = +0.0369  ← 几乎无用
    消融2/5 (+_and)  → Δ = +0.0109  ← 更小!
    消融3/5 (+_d)    → Δ = +0.0518  
    消融4/5 (+_s)    → Δ = +0.2574  ← ★ 跳升! 关键子token
    消融5/5 (+pend)  → Δ = +0.1992  ← 完整效应

    
    效应分布图:
    Ġr    _and   _d    _s    pend
    ▏     ▏      ▏     ██▌   ██
    7%    2%    10%    52%   40%  (占全效应的比例)
```

### 3.2 核心发现

**r_and_d_spend 的信息高度不均匀分布**：

- 前 3 个子 token (`Ġr`, `_and`, `_d`) 合计贡献不到 20%
- 第 4 个子 token (`_s`) 贡献了 52%——最大的单一贡献者
- 最后子 token (`pend`) 贡献 40%
- **关键突破在 `_s` 和 `pend`——这两个子 token 区分了 "r_and_d_spend" 与其他可能的 "r_and_d_..." 前缀**

### 3.3 为什么信息集中在后半段？

```
Qwen2.5 tokenizer 的 BPE 分词:
"r_and_d_spend" → [Ġr, _and, _d, _s, pend]

在 prompt "JetCircuit..." 的上下文中，
Ġr 可能出现在很多词中 (report, region, ...)
_and, _d 同样是高频前缀
只有 _s + pend 组合才唯一标识 "spend"

→ L0 的编码能力受 tokenization 粒度影响
→ 越碎片的 token，信息越被稀释到多个位置
→ 关键区分信息集中在尾部的独特性子 token
```

### 3.4 跨 token 统一模型

```
Token                     │ Sub-tokens │ L0 Δ    │ per-sub Δ │ 信息密度
──────────────────────────┼────────────┼─────────┼───────────┼─────────
funding                   │     1      │ +0.399  │  +0.399   │ ★★★
r_and_d_spend             │     5      │ +0.199  │  +0.040   │ ★
total funding             │     2      │ +0.059* │  +0.030   │ ★
──────────────────────────┼────────────┼─────────┼───────────┼─────────
* P27 full-layer mean, 非 combined

规律: sub-token 越多 → 信息越稀释 → L0 效应越小
```

## 4. P35: L16 Decomposition — 为什么 L16 是次高峰？

### 4.1 实验结果

```
Sample │ L16 self_attn │ L16 mlp  │ L16 combined │ dominant
───────┼───────────────┼──────────┼──────────────┼──────────
  4    │    -0.0142    │ +0.0690  │   +0.0588    │  mlp
 14    │    -0.0062    │ +0.0781  │   +0.0730    │  mlp
 24    │    -0.0047    │ +0.0715  │   +0.0685    │  mlp
───────┼───────────────┼──────────┼──────────────┼──────────
 MEAN  │    -0.0084    │ +0.0729  │   +0.0668    │  mlp (100%)
```

### 4.2 判决

```
L16 self_attn:  -0.008 (微负, 3/3 负)
L16 mlp:        +0.073 ★ (正, 3/3 正)
MLP 贡献占比:   100%

→ L16 的次高峰完全来自 FFN (mlp)
→ self_attn 不仅不帮助，甚至有轻微的负面贡献
```

### 4.3 这是什么？

L16 是 FFN 知识检索阶段：

```
L0:        嵌入 → 残差编码      (信息入口)
L1-L15:    残差流传播、语义丰富化
L16:       FFN 知识检索 ★       (将 token 表示与存储知识对比)
            ↑ self_attn 此时不再贡献——信息已经在残差流中
            ↑ FFN 查找 "funding" → "回答总融资额需要什么知识？"
L17-L23:   生成准备
```

**L16 的 FFN 不是在编码 token，而是在检索知识。** 这就是为什么：
- Combined ablation 的 L16 Δ 只有 +0.07（远小于 L0 的 +0.34）
- self_attn 在此处为负（信息检索不需要更多上下文整合）
- 它在 P30 full-layer 曲线上表现为次高峰

## 6. 四个"为什么"的答案

| 问题 | 答案 |
|------|------|
| 为什么是 L0？ | L0 是 embedding→residual 的唯一传导器。Δembed=ΔL0，天花板效应证明 L0 不产生新信息，只是传递 |
| 为什么 L1-L3 不编码？ | L1-L3 接收的是 L0 已经打包好的残差表示。它们传播和混合，但不重新编码。Head 1 在 L0 已经完成 100% 自聚焦 |
| 为什么 r_and_d_spend 效应更小？ | 5 个子 token 稀释了信息浓度。关键区分信息集中在 `_s`+`pend`（后 40% 子 token），前 60% 子 token 几乎无贡献 |
| 为什么 L16 是次高峰？ | L16 是 FFN 知识检索阶段（100% mlp 贡献，self_attn 为负）。检索"funding"→"需要什么知识来回答总融资额" |

## 7. 最终模型

```
P26→P31→P35 知识链条:

P26: full-layer L3 peak         → 发现 L3 处累积残差最强
P27: L3 不通用 (token-dependent) → r_and_d_spend 在 L4
P28: first-mention effect       → 首现位置决定编码层
P29: 子模块拆分 → L3 消失        → L3 peak 是组合 artifact
P30: combined 消融 → L0 40x 碾压 → L0 是真正的信息源
P31: cross-token 复现            → 一般规律，非 funding 特例
P32: embedding ≈ L0              → L0 是管道，不是编码器
P33: Head 1 完美自聚焦           → 发现 L0 的编码机制
P34: tokenization 稀释           → 解释了效应强度差异
P35: L16 = FFN 知识检索          → 区分了编码 vs 检索阶段
```
