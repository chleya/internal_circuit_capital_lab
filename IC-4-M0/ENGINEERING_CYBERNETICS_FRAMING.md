# 工程控制论框架：小模型结构适应性研究

> **一句话总框架**
>
> 我们将小规模语言模型视为一个高维、非线性、对输入组织高度敏感的受控对象，研究其内部状态的可观测性、结构漂移、以及闭环补偿机制。
>
> 借用钱学森《工程控制论》的视角：面对一个结构极度复杂、存在大量不确定性和噪声的系统，控制的本质在于利用反馈克服系统内部的不确定性与噪声，而非穷举每一个微观机制。

---

## 1. 为什么需要这个框架

### 1.1 叙事困境

如果我们的项目被表述为：

> "我们也能做 activation steering / gate / 检索增强"

那么它和 Self-RAG、Representation Engineering、DoLa 等工作处于同一层级——容易被看成**已有路线的小模型复现**。

但我们的实验已经走到了一个不同的位置：

- 我们关心的不是"能不能做"，而是 **"为什么小模型明明有潜在能力，却默认不能把它组织成稳定行为"**
- 我们拆出了三个瓶颈（吸收/稳定/组织），而不是仅关注一个工程技巧
- 我们发现方向特异性不存在（P2）、consolidation 产生结构性坏资本（IC-2c）、谄媚需要方差坍塌而非均值偏移（M7）

**我们需要一个能把所有这些发现组织起来的顶层框架。**

### 1.2 工程控制论为什么对

它不是硬套，而是和我们的实验事实贴合：

| 我们的发现 | 控制论解读 |
|---|---|
| M7-Lv2: 模型内部存在 latent verification capability | 系统存在**可观测状态信号** |
| M3-v6: probe→gate→hook 管线将 H 从 0.867 降到 0.667 | 这是一个最小**闭环反馈控制系统** |
| IC-2c: consolidation 导致 centroid drift + purity collapse | 无锚定输入下的**系统失稳与渐进式漂移** |
| Position Sensitivity: KNN=1.0 | 输入组织方式是一个**强时变扰动** |
| P2: v_hall = v_orthogonal, 方向特异性不存在 | 系统对**能量扰动敏感，但对方向不敏感** |

这个框架**不是比喻，是真实的结构对应。**

## 2. 概念映射表

| 我们的实验术语 | 《工程控制论》定义 | 本质解释 |
|---|---|---|
| **0.5B 模型隐空间 (896D)** | **状态空间 (State-Space)** | 描述系统运行轨迹的高维数学坐标系 |
| **Token 输入流 / 位置敏感 (KNN=1.0)** | **系统扰动与输入噪声 (Disturbance)** | 输入位置的微小变化对状态空间造成巨大扰动 |
| **Latent Verification (M7-Lv2)** | **内部可观测状态 (Observable State)** | 系统内部已产生正确信号，说明系统**部分可观测** |
| **输出端幻觉 / 谄媚** | **稳态误差 / 开环失控 (Steady-State Error)** | 默认状态下控制路径错误，输出偏离期望参考信号 |
| **M3-v6 门控管线** | **闭环反馈控制系统 (Closed-Loop Feedback)** | 探针采样→误差计算→控制输入→纠正偏差 |
| **IC-2 质心漂移 + 纯度崩溃** | **无锚定漂移 (Unanchored Drift / Instability)** | 连续压缩下缺乏参考锚点，状态轨线滑向不稳定区域 |
| **P2 正交向量同等效果** | **能量扰动效应 (Energy Perturbation)** | 不是"方向控制"，而是改变系统能量耗散状态 |

## 3. 三个子问题：可观测 → 可控 → 稳定

### 3.1 可观测性 (Observability)

> **模型内部是否存在与更好行为对应的状态信号？**

**Solid ground:**
- M7-Lv2: latent verification capability 存在。`fact_checker` prompt 将 sycophancy 从 0.60 降到 0.40 (-20pp)
- M7-D: 谄媚是预训练先验（Base 100%, Instruct 96.7%），不是 RLHF 产物
- Oracle routing 正确率 85.7% — 模型"能做但不能迁对"
- T1/T2: hallucination 和 sycophancy 在 prefill 阶段即可被线性探针分离

**未证明的:**
- 整体可观测性。当前只证明**存在与目标行为相关的部分可观测状态信号**
- 信号的充分性：是否仅凭当前可观测信号就能完整重建最优行为轨迹？

### 3.2 反馈控制 (Feedback Control)

> **能否在不改权重的前提下，通过状态读出、门控和小幅控制输入，减小输出偏差？**

**Solid ground:**
- M3-v6 是经过验证的最小闭环系统：
  - 状态采样：`last_prompt_token`
  - 误差估计：`logistic probe` (train_acc=1.0, cv_acc=1.0)
  - 控制器：`hard gate` (threshold=0.5)
  - 控制输入：`hook injection` (α=−1.0)
  - 结果：H 从 0.867 → 0.667，C 保持 0.600 不变
- M4: OOD 条件下仍然鲁棒（3 scenarios × 3 alphas 全部通过）
- 超过 controls: real gate (H=0.667) 显著优于 random gate (H=0.933) 和 shuffled gate (H=0.800)

**未证明的:**
- 精确控制方向尚未建立。P2 证明 v_hall 和 norm-matched orthogonal 效果相同 (−0.283 vs −0.283)
- 当前的"控制"更接近**扰动能量的系统级调节**，而非方向特异性的精确控制
- 可用李雅普诺夫稳定性/吸引子语言作为**工作假说**，但无严格数学证明

### 3.3 稳定性 (Stability)

> **连续压缩、整合、位置变化是否导致状态轨线漂移、结构坍塌？**

**Solid ground:**
- IC-2c: Consolidated 的 best_action_match (0.115) < Random (0.333) — consolidation 产生了系统性坏资本
- IC-2c.1 根因拆解：跨分布平均化 + centroid 失衡 (2.86→7.27) + centroid drift (~1.0-1.3/step)
- Topology Audit: TPR=0.875 (拓扑保留) 但 Purity=0.261 (全部 20 个 centroid 跨 seed 严重混合)
- 破坏主要是聚类混合而非拓扑崩坏
- Position Sensitivity: Rep Shift KNN=1.0 (相同内容不同位置→完全不同的 hidden state)

**未证明的:**
- 稳定性条件的精确刻画（何时漂移、何时不漂移、阈值在哪）
- 锚定机制的具体设计
- ASFT (arXiv:2509.23753) 提供了 KL 锚定的理论可能，但尚未在我们的设置中验证

## 4. 闭环反馈控制流图 (M3-v6)

```
                          【闭环伺服控制流】
                          
输入扰动 (Position/Prompt) ──► [ 受控对象: 0.5B 隐空间 ] ───► 输出 (带幻觉)
                                    │                ▲
                            状态采样 (last_token)       │ 控制输入 (α·v)
                                    ▼                │
                              [ 逻辑探针 ] ──► [ 硬门控控制器 ]
                               (误差估计)      (决策: inject? skip?)

                    若 P(unanswerable) ≥ 0.5 →
                      注入 steering vector
                      将输出推向 abstention
```

这个流图与传统的"prompt engineering"或"activation addition"的关键区别在于：
- **不是开环**（固定 prompt 或固定向量）
- **是闭环**（基于实时状态采样做出控制决策）
- **不修改模型权重**（相当于在已有 Plant 上加了一个外挂 Controller）

## 5. 三瓶颈的控制论翻译

| 瓶颈 | 原始定义 | 控制论翻译 |
|---|---|---|
| **A: Absorption** | 输入碎片化导致信息丢失 | 输入如何进入状态空间；输入扰动如何扭曲内部状态轨线 |
| **B: Stabilization** | 跨样本压缩导致结构漂移 | 状态轨线如何在无锚定更新中保持稳定，不因连续压缩而发散 |
| **C: Organization** | 能力存在但路由不通 | 内部已有信号如何被正确路由到输出行为（控制回路是否正确闭合） |

这一翻译的工程价值：
- **IC-4** = Organization 补偿（闭环控制回路的构造）
- **IC-2** = Stabilization 诊断（结构漂移的定量病理）
- **Position Sensitivity** = Absorption 诊断（输入组织方式对状态空间扰动强度的定量）
- **P2** = 系统非线性响应边界的探测（方向特异性的证伪 = 系统对控制方式的约束）

## 6. 与已有工作的关系：攻击面的重新定位

> **我们关心的不是"有没有一个检索开关"，而是"小模型作为高维受控对象，为何在输入扰动、结构漂移和能力路由上失稳；我们如何用反馈和锚定机制补偿这种结构适应缺陷"。**

| 已有工作 | 与原项目的关系 | 在工程控制论框架下 |
|---|---|---|
| **Self-RAG** | 高层结构最像（probe+gate+retrieval） | Self-RAG 在系统外部挂了一个"外呼开关"。我们在系统内部 896D 状态空间里做状态采样、误差计算、反馈注入。层级不同。 |
| **Representation Engineering / ActAdd** | 强相关。我们站在这条线后面。 | 他们止于"向量能控行为"的工程验证。我们在问"为什么小模型 controllability 会失真"，并拆出 routing/trajectory/sample construction artifact。 |
| **DoLa** | 方法邻近（层间对比解码） | DoLa 是实用的解码技巧。我们是机制工程——不只要一个技巧，要一套可推广的控制程序。DoLa 是必须比较的 baseline，但不是理论替代。 |
| **Toolformer** | 工程范式邻居 | Toolformer 学"调用外部工具"。我们研究"为什么模型内部不能稳定组织已有的能力/信息"。机制层不同。 |

## 8. 需要守住边界的地方（不能说过头）

| # | 边界 | 更稳的表述 |
|---|---|---|
| 1 | "可观测" | **存在与目标行为相关的部分可观测状态信号**，尚未证明系统整体可观测 |
| 2 | "可控" | **已观察到 early-state 对外部扰动敏感，但方向特异性尚未建立** (P2 是关键刹车) |
| 3 | "李雅普诺夫稳定性 / 吸引子盆地" | 可作为动力学解释框架的**工作假说**，当前已有实验事实与之相容，但未构成严格数学证明 |
| 4 | "输入噪声" | Position sensitivity 是**结构化输入扰动**（位置/排列导致的系统性扰动），不是完全随机噪声 |
| 5 | Scale 泛化 | 所有实验在 **0.5B 模型**上完成，尚未在 1.5B/7B 上复现 |
| 6 | 任务泛化 | 使用**合成 QA 数据**（30A+30U），尚未在自然分布数据上验证 |

## 9. 推荐对外口径

### 9.1 一句话定调

> 我们借用工程控制论的视角，将小语言模型视作一个高维非线性受控对象。本研究不试图穷举模型内部所有微观机制，而是聚焦于：1) 内部状态中是否存在可观测的有用信号；2) 这些信号为何在输入扰动与结构整合中失稳；3) 是否可以通过不改权重的闭环补偿机制，降低输出端的稳态误差。

### 9.2 项目主叙事（提案/答辩/论文导言版）

> 钱学森先生的《工程控制论》教会我们：面对复杂的受控对象，控制的本质在于利用反馈克服系统内部的不确定性与噪声。
>
> 本研究摒弃了传统大模型"扩大参数规模"的粗放路线，将小模型（0.5B）视为一个高维非线性时变受控对象——
>
> - 我们定量诊断了系统在信息吸收与稳定过程中的状态发散病理（IC-2：无锚定质心漂移与流形纯度崩溃）
> - 在理论上证伪了传统表征工程中"线性方向特异性"的控制假设（P2：提出了"全局能量扰动"的动力学解释）
> - 设计了一套创新的内部状态闭环反馈伺服系统（IC-4 门控管线），在不改动模型原始权重的条件下，动态抑制了系统内部的稳态误差（幻觉率 -20pp）
>
> 这**不是 Self-RAG 的变体**——Self-RAG 是在系统外部挂一个"检索开关"。我们在系统内部 896 维状态空间里做状态观测、误差估计和反馈控制。

### 9.3 一句话自我定位

> **我们不是在做一个低配版 Self-RAG。我们在研究小模型作为高维非线性受控对象的可观测性、可控性边界和结构稳定性——这是一套小模型机制控制理论框架的构建工程。**
