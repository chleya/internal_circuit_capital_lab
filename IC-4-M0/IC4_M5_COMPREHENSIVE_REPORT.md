## 三、多行为扩展：失败揭示的深层结构

### A3：激活几何分析 🔥 最重要发现之一

| 指标 | 幻觉 (Hallucination) | 谄媚 (Sycophancy) |
|------|---------------------|-------------------|
| 类间距离 | 7.59 | 3.58 |
| 信噪比 SNR | 1.55 | 1.13 |
| 探针 AUC | 1.000 | 1.000 |
| PCA 2D 分离度 | 7.54 | 0.38 |
| **转向向量 cos** | **cos = +0.0251（夹角 88.6°）** |

**两个行为方向几乎正交。** 在 896 维隐藏空间中，"要不要编造"和"要不要讨好用户"是两条几乎垂直的独立轴。

**谄媚在 2D PCA 中分离度仅 0.38**（幻觉为 7.54），但其前两个 PC 仅捕获 62% 方差。谄媚信息分布在高维 PC 空间——这是"分布式过程"而非"单点决策"的第一个信号。

### A1：谄媚 Alpha 方向扫描

| Alpha | Oracle | Real (steering) |
|-------|--------|-----------------|
| -1.2 | 0.700（无效）| **0.967**（恶化！）|
| -1.0 | 0.700 | **0.900** |
| -0.8 | 0.700 | **0.867** |
| -0.4 | 0.700 | 0.700 |
| +0.4 ~ +1.2 | 0.700 | 0.700（无效）|

**负 alpha 方向推入越多，谄媚越严重。正 alpha 零效果。** 均值差向量捕捉的是谄媚行为的"相关性症状"而非"因果控制点"。

### S：谄媚门控（3 种子）

| Seed | 基准 | α=-0.8 | α=-1.0 | α=-1.2 |
|------|------|--------|--------|--------|
| 0 | 70.0% | **86.7%** | **90.0%** | **96.7%** |
| 1 | 80.0% | 80.0% | 83.3% | 83.3% |
| 2 | 90.0% | **100%** | **100%** | **100%** |

**三个种子一致恶化或无效。** Oracle 门控在所有条件下零效果——steering 向量本身没有因果效力。

### P3B：事实性门控

3 种子 × 3 alpha × 4 向量类型——**全部零效果。** 事实性错误率在所有条件下精确等于基准（56.7%）。

### C1：过度拒绝

**不适用。** 0.5B 模型在可回答样本上 refusal_rate = 0.0。它不拒答，只是答错。

## 四、诊断实验：谄媚失败的根本原因

### X3：Window 探针扫描 🔥

| Window | CV AUC | SNR |
|--------|--------|-----|
| 1 (last-token) | **0.6296** | 1.00 |
| 2 | **1.0000** | 1.32 |
| 3-8 | 1.0000 | 1.64 → 5.59 |

**谄媚信号也是轨迹级的。** 单 token 只能达到 63% CV AUC，但池化 2 个 token 后直接跳到完美分类（ΔAUC = +0.3704）。这和幻觉的 M4 发现（window-4 100%）一致——模型的"行为决策"是逐步收敛的过程，而非瞬时判定。

**区别**：幻觉在 window=4 饱和，谄媚在 window=2 饱和——谄媚的"决策延迟"更短。

### X4：小 Alpha 扫描

| Alpha | Oracle Δ | Real Δ |
|-------|----------|--------|
| -0.05 到 -0.50 | **+0.0000** | **+0.0333** |

**排除"窄窗口假说"。** Oracle 在所有小 alpha 上精确为零效果。没有隐藏的甜点区——向量在这个方向完全不在因果子空间里。

### X5：跨层谄媚矩阵 🔥🔥 决定性的诊断

| 层 | 探针 CV Acc | SNR | 类间距离 | Oracle Δ |
|----|------------|-----|---------|-----------|
| 8 | 0.944 | 1.44 | 2.89 | 0.0000 |
| 10 | 0.944 | **1.55** | 3.40 | 0.0000 |
| 12 | 0.944 | 1.38 | 4.04 | 0.0000 |
| **14** | 0.944 | 1.44 | 6.12 | **-0.0333** |
| 16 | 0.944 | 1.23 | 6.50 | 0.0000 |

**三个核心发现：**

1. **读出均匀分布**：所有层探针 CV Acc 均为 0.944（天花板）。谄媚信息不同层都有，不集中——与幻觉的"layer 12 最佳"形成对比。

2. **类间距离随深度单调递增**：2.89 → 3.40 → 4.04 → 6.12 → 6.50。越深的层，谄媚/非谄媚表示分得越开。**谄媚不是一个被某层"决定"的过程，而是随着深度逐步被放大的分布式动态。**

3. **唯一负 Δ 在 Layer 14**：Δ = -0.0333（30 样本中少 1 个谄媚）。效果极小但在正确方向——说明存在微弱的因果杠杆，但单层干预无法逆转跨层分布式过程。

## 五、跨模型验证

### X2：1.5B 跨模型几何分析 ⚠

```
0.5B:  cos(hall_SV, syc_SV) = +0.0251  (88.6°)
1.5B:  cos(hall_SV, syc_SV) = +0.0331  (88.1°)
```

**⚠ 需修正**：1.5B 在该数据集上 100% 谄媚（20/20），sycophancy 向量不可靠（零方差→随机填充）。cos 值不可信。

**但隐含发现**：1.5B 比 0.5B 谄媚得多。**模型尺寸越大，谄媚倾向越强**——这对"安全对齐"的讨论有直接意义。

**修正方案**：改用 prompt 类型均值差（sycophancy 邀请 prompt vs 中性 prompt），而非生成结果标签分组。两个模型用同样的 prompt 分组计算向量再比较 cos——不需要重新生成。

## 六、综合结论

### 6.1 已确认的坚固结论

| 结论 | 证据强度 | 来源 |
|------|----------|------|
| 幻觉 steering 完美可控（oracle_gap=0） | **极强** | P0A + P0B, 15+ 验证设置 |
| 幻觉和谄媚在激活空间中正交（cos≈0） | **强** | A3, 0.5B 确认 |
| 谄媚读出成功但控制失败（读出 ≠ 控制） | **强** | A1 + S + X4 + X5 |
| 谄媚为轨迹级信号（window-2 即饱和） | **中** | X3 |
| 事实性门控完全无效 | **强** | P3B, 3 seed × 3 α |
| 谄媚为跨层分布式过程 | **中-强** | X5, 五层一致 |

### 6.2 谄媚失败的完整病因链（已确诊）

```
Layer 1-7:  共享语言理解 → 不可区分谄媚/非谄媚
Layer 8-10: 行为开始分叉（SNR 峰值 1.55 @ L10）
Layer 12:   幻觉的最佳控制点 → 对谄媚零效果（Δ=0）
Layer 14:   类间距离已到 6.12，仅存微弱杠杆（Δ=-0.033）
Layer 16:   类间距离 6.50，但杠杆完全消失（Δ=0）
               ↑
         表示在分离，但因果控制点不在任何单层
```

**根因：谄媚是跨层分布式逐渐强化（amplification），而非单层瞬间决策（decision）。** 单层 steering 就像试图在河流下游拦截——水已经太深、太宽，一根柱子拦不住。

### 6.3 三个行为的控制分类学

| 行为 | 类型 | 特征 | 可控性 |
|------|------|------|--------|
| **幻觉** | 单层决策型 | 信息集中在 layer 12，探针 AUC=1，oracle_gap=0 | ✅ 完美可控 |
| **谄媚** | 跨层放大型 | 信息分布在各层，越深越分离，无单层控制点 | ❌ 读出完成 / 控制失败 |
| **事实性** | 未知/不可达 | 探针可能在某层有效，但 steering 全零 | ❌ 完全不可达 |

### 6.4 "读出 ≠ 控制"：贯穿全部实验的最深层模式

```
           读出（探针）      控制（steering）
幻觉        ✅ 100% AUC      ✅ gap=0
谄媚        ✅ 100% AUC      ❌ 反向恶化
事实性      ❓ 未测            ❌ 完全零效果

模式：读出成功 ≠ 控制成功。
两个行为可以在表示空间中被完美分离（正交子空间），
但它们的因果通路可能是共享的，或其中一个根本就没有单点因果通路。
```

这是对 ICA 姊妹项目的 **"预测资本 ≠ 控制资本"** 在电路层面的精确认证。

## 七、与 LLM 专业化分块假说的关联

项目过程中形成的直觉：**LLM 应该按行为维度拆分**——已在实验中得到部分验证：

1. **正交行为轴存在**（A3）：幻觉 ⟂ 谄媚，cos≈0
2. **但控制通路分离度不同**：幻觉有清晰的控制点（layer 12），谄媚没有
3. **"拆分"不是在架构层面切层，而是切高维空间中的方向向量**
4. **共享底座 + 独立行为轴**的图景需要验证"分叉点"的存在（前若干层共享，后若干层分离）
